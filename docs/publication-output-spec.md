# Publication Output Spec

Date: 2026-05-09

## Purpose

This document defines how QuarterlineV2 assembles and exports quarterly market
reports from a desktop workspace.

## Current Status

No V2 publication renderer or export flow exists. The target report structure
is informed by the CBRE Atlanta Office Q1 2026 reference report (see
`docs/reference-artifacts/`).

## Reference Report Analysis

The CBRE Atlanta Office Q1 2026 Figures report
(`docs/reference-artifacts/Atlanta_Office_2026_Q1_Figures.pdf`) is an 8-page
quarterly market report. Its structure defines the quality bar:

### Page 1: Cover / Market Overview

- Header: property type, market, quarter.
- Headline: plain-language market narrative title.
- Key metrics banner: 5 headline numbers with directional arrows (availability
  rate, net absorption SF, construction delivered SF, under construction SF,
  asking lease rate $/SF).
- Market overview narrative: 2 paragraphs of analyst-written market context.
- Figure 1: Historical Net Absorption, Deliveries, and Vacancy (combo
  bar/line chart, quarterly, ~3 years).

### Pages 2-4: Topical Sections

Each section pairs narrative text (left column) with one or two charts (right
column):

- **Availability**: narrative + Figure 2 (Availability Rates by Class, line
  chart) + Figure 3 (Average Direct Asking Rate by Class, line chart).
- **Net Absorption**: narrative + Figure 4 (Net Absorption Trend, bar chart
  with Class A vs. other) + Figure 5 (Construction Activity, area/bar chart).
- **Leasing Activity**: narrative + Figure 6 (Leasing Activity Trend, stacked
  bar chart) + Figure 7 (Leasing by Submarket, pie/donut chart) + Figure 8
  (Key Lease Transactions, table with tenant/SF/type/address/submarket).

### Pages 5-7: Statistical Tables

Dense data tables with consistent column structure:

- Figure 9: Suburban Market Statistics by Class.
- Figure 10: Urban Market Statistics by Class.
- Figure 11: Metro Market Statistics by Class.
- Figure 12: Market Statistics by Submarket.

Standard columns across all tables: Property Class or Submarket, Net Rentable
Area (MSF), Total Vacancy (%), Total Availability (%), Direct Availability (%),
Sublease Availability (%), Avg Direct Asking Rate ($/SF FSG/yr), Current
Quarter Net Absorption (SF), YTD Net Absorption (SF), Deliveries (SF), Under
Construction (SF).

### Page 8: Market Area Overview

- Submarket boundary map (color-coded by submarket).
- Definitions glossary.
- Survey criteria.
- Analyst contacts.

## Report Structure Target

QuarterlineV2 reports should follow a similar structure:

1. **Cover page**: market, property type, quarter, headline, key metrics
   banner.
2. **Topical sections**: each pairs a narrative with one or more exhibits
   (charts, tables, maps). Sections are assembled from pinned modules.
3. **Statistical tables**: dense submarket and class breakdowns with consistent
   column schemas.
4. **Market map**: submarket boundary visualization.
5. **Appendix**: definitions, data sources, methodology notes.

## Report Output Goals

- Analysts can pin modules into a report queue from any workspace view.
- Pinned modules include their data context (chart data, table query, map
  bounds) so the report is reproducible.
- Report output is clean and client-readable, separated from internal review
  context.
- Claims should be traceable to evidence, data, or analyst judgment.
- Narratives can be authored in-app, generated by built-in AI, or written by
  external AI tools via the workspace bridge.
- Export works entirely from the local desktop app.
- Export artifacts are reproducible from a saved workspace snapshot.

## Candidate Output Types

- Local PDF (primary target for MVP).
- Local HTML report package.
- Presentation/deck export (future).
- Internal review packet with tracked changes and notes (future).

## Narrative Authoring

Report section narratives live in the workspace's `narratives/` folder as
markdown files. They can be created through three paths:

1. **Manual**: analyst writes directly in the app's report editor.
2. **Built-in AI**: the app generates a draft from market data and evidence
   using a configured AI provider.
3. **External AI bridge**: an external tool (Claude Desktop, Codex) reads the
   workspace data files and writes narrative markdown to `narratives/`.

The app assembles the final report from these narratives plus pinned exhibits.
