# AI Bridge Specification

Date: 2026-05-09

## Purpose

This document defines how external AI tools (Claude Desktop, Codex, or any
agent that can read a folder) interact with a QuarterlineV2 workspace. The
bridge is a folder-structure contract: the workspace stores key content in
human-readable files so AI tools can read context and generate content without
needing the app to be running.

## Workspace Folder Structure

Each workspace lives at:

```
~/.quarterline/workspaces/<workspace-id>/
```

Contents:

```
<workspace-id>/
├── WORKSPACE.md              # AI-readable manifest
├── workspace.db              # SQLite database (app-managed, do not edit)
├── data/
│   ├── market-statistics.json
│   ├── submarket-statistics.json
│   ├── property-data.json
│   ├── lease-data.json
│   ├── financial-metrics.json
│   └── scenario-results.json
├── narratives/
│   ├── market-overview.md
│   ├── availability.md
│   ├── asking-rent.md
│   ├── net-absorption.md
│   ├── construction-activity.md
│   ├── leasing-activity.md
│   └── custom/
│       └── <analyst-named-sections>.md
├── notes/
│   ├── analyst-notes.md
│   └── <topic>.md
├── sources/
│   └── <ingested files>      # Confidential — do not read or expose
├── exports/
│   └── <generated reports>   # App-managed
└── .quarterline/
    └── config.json           # Workspace settings (app-managed)
```

## WORKSPACE.md Manifest Format

This file is the entry point for any AI tool opening the workspace folder. It
describes the workspace context and tells the AI what it can and cannot do.

```markdown
# Quarterline Workspace

## Context

- **Workspace**: <workspace name>
- **Market**: <market name, e.g., "Atlanta Office">
- **Quarter**: <current quarter, e.g., "Q1 2026">
- **Property Type**: <e.g., "Office">
- **Last Updated**: <ISO date>

## What You Can Do

You are an AI assistant helping a commercial real estate research analyst
assemble a quarterly market report.

### Read (for context)

- `data/market-statistics.json` — market-level statistics by property class.
- `data/submarket-statistics.json` — submarket-level statistics.
- `data/property-data.json` — property and asset details.
- `data/lease-data.json` — lease records with tenant, SF, terms.
- `data/financial-metrics.json` — NOI, cap rates, asking rents, absorption.
- `data/scenario-results.json` — what-if scenario outputs.
- `notes/*.md` — analyst notes and observations.
- `narratives/*.md` — existing report section drafts.

### Write (to generate content)

- `narratives/*.md` — write or update report section narratives.
- `notes/*.md` — add analysis notes.

Use the section naming convention below when creating new narrative files.

### Do Not Touch

- `workspace.db` — SQLite database, managed by the app.
- `sources/*` — confidential source documents.
- `exports/*` — generated report files.
- `.quarterline/*` — app configuration.
- `WORKSPACE.md` — this file (app-managed).

## Report Structure

The quarterly report follows this section order. Each section corresponds to
a narrative file in `narratives/`:

1. `market-overview.md` — headline market narrative.
2. `availability.md` — availability trends and analysis.
3. `asking-rent.md` — rent trends by class and submarket.
4. `net-absorption.md` — absorption patterns and drivers.
5. `construction-activity.md` — pipeline and deliveries.
6. `leasing-activity.md` — leasing volume, key transactions.
7. `custom/<name>.md` — additional analyst-defined sections.

## Writing Guidelines

- Write as a CRE research analyst, not as an AI assistant.
- Be specific: cite numbers from the data files.
- Use the market and quarter from the Context section above.
- Refer to submarkets, property classes, and tenants by name.
- Keep narratives to 2-3 paragraphs per section.
- Do not fabricate data. If a data point is not in the data files, do not
  invent it.
- Use parentheses for negative numbers: (443,000) not -443,000.
- Use SF for square feet, MSF for millions of square feet, $/SF for per
  square foot rates.
```

## Narrative File Format

Each narrative file in `narratives/` is plain markdown:

```markdown
# Market Overview

The Atlanta office market showed mixed but improving signals in Q1 2026.
Net absorption declined by 443,000 sq. ft., impacted by large blocks of
sublease space transitioning back to direct availability. Excluding these
rollovers, much of the market posted modest gains.

The availability rate declined to 33.1%, down 1.8 percentage points
year-over-year. Cumberland/Galleria, Midtown, and North Fulton led the
improvement where tenant demand is most active.

Rents remained well supported, with average asking rents reaching $33.75
per sq. ft., up 1.4% year-over-year.
```

No frontmatter, no metadata. The app associates narrative files with report
sections by filename convention. The analyst or AI tool writes the narrative
content; the app handles layout, exhibits, and assembly.

## External Change Detection

When the app regains focus or the analyst triggers a refresh:

1. The app checks modification timestamps on files in `narratives/` and
   `notes/`.
2. If any file has been modified externally since the last app-managed save,
   the app shows a notification: "External changes detected in
   narratives/availability.md. Import?"
3. The analyst can preview the diff and accept or reject each change.
4. Accepted changes are imported into the report draft state.
5. Rejected changes remain in the file but are not reflected in the report
   until the analyst accepts them.

## Security Boundaries

- `sources/` is never included in the AI bridge readable set. Source files
  may contain confidential client documents, lease abstracts, or proprietary
  data. The app extracts relevant data points into the `data/` JSON files,
  which contain only aggregated or anonymized market-level information.
- The `WORKSPACE.md` manifest explicitly instructs AI tools not to access
  `sources/`, `workspace.db`, `exports/`, or `.quarterline/`.
- The app does not expose full file paths in exported JSON (property IDs are
  opaque identifiers, not filesystem paths).
- If the analyst enables encryption for a workspace, `sources/` files are
  encrypted at rest and `data/` JSON exports contain only what the analyst
  has approved for the AI bridge.
