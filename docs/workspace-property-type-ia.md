# Workspace And Property-Type IA

Date: 2026-05-09

## Purpose

This document will define how V2 organizes workspaces, portfolios, markets,
properties, quarters, and property-type templates.

## Current Status

No V2 IA has been implemented.

## IA Questions

- What is the first screen of the desktop app?
- Is the workspace centered on a portfolio, market, property, or report?
- How does Market Level differ from Property Level?
- How should property types change labels, fields, and report sections?
- How does an analyst switch quarters?
- How does the app show local save/sync/export state?

## Candidate Workspace Axes

- Portfolio.
- Market.
- Property.
- Property type.
- Quarter.
- Report template.
- Scenario set.
- Report assembly queue.

## Design Target

The concept artwork suggests a persistent portfolio sidebar with global
Market-Level and Property-Level filters. Future IA should validate or revise
that structure before implementation.
